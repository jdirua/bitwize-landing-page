## Blueprint Handling Protocol

- **Always Prompt First:** Before making any changes to `blueprint.md`, I must first inform the user of the intended changes and receive their explicit approval.
- **No Overwriting or Recreation:** I must not overwrite or recreate the `blueprint.md` file. 
- **Targeted Updates Only:** My modifications will be limited to updating the specific section I am currently working on or appending new information to existing sections as required.
- **Approval is Mandatory:** I am not to touch the file for any reason unless I have explicit user consent.
